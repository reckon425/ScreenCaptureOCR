# 多源翻译工具（ScreenCaptureOCR）— 项目策划文档

## 1. 项目概述

本项目是基于 Windows Forms 的桌面翻译工具，支持三种文字输入方式（手动输入、截图OCR识别、文件上传），统一汇聚到翻译引擎进行多语种互译，并提供翻译历史管理、结果导出和系统托盘后台运行等配套功能。用户可随时通过全局热键截图获取屏幕文字，也可直接输入或上传文件完成翻译。

## 2. 需求分析

### 2.1 用户场景
- 阅读外文文档或软件界面时，需要快速获取文字翻译
- 无法直接复制文字的图片、PDF 或扫描件中的内容需要提取
- 批量处理文字文件（.txt、.docx、.pdf）的翻译需求
- 保留历史翻译记录以便回顾和对比

### 2.2 功能需求

| 模块 | 功能 | 说明 |
|------|------|------|
| 手动输入翻译 | 文本框直接输入文字 → 一键翻译 | 支持粘贴，自动语言检测 |
| 截图OCR翻译 | 全屏截图 → 区域选择 → OCR识别 → 翻译 | GDI+ 遮罩-挖空选区交互 |
| 文件上传翻译 | 上传文件自动解析 → 填入文本框 → 翻译 | 支持 .txt/.docx/.pdf |
| 翻译引擎 | 多语种互译 | Google Translate，7种目标语言 |
| 历史记录 | 自动保存每次翻译结果 | 标记来源类型（截图/手动/文件），正文+译文完整保存 |
| 结果导出 | 翻译结果保存为 .txt | 包含原文和译文 |
| 系统托盘 | 后台运行 | 最小化到托盘，全局热键随时唤醒 |

## 3. 技术选型说明

| 技术 | 用途 | 选型理由 |
|------|------|----------|
| C# / .NET Framework 4.8 | 开发平台 | 课程核心技术栈，WinForms 原生支持 |
| WinForms | 桌面 UI | 开发效率高，控件事件驱动模型成熟 |
| Google Translate API | 文字翻译 | 免费接口，自动语言检测，无需注册 API Key |
| 百度 OCR API | 图片文字识别 | 免费额度充足，识别准确率高 |
| GDI+ | 截图与选区交互 | 屏幕像素级操作能力，CopyFromScreen + 分层绘制实现选区 |
| PdfPig | PDF 解析 | 纯 .NET 开源库，无需额外依赖 |
| Newtonsoft.Json | 数据序列化 | .NET 生态标准 JSON 库 |

**差异化优势**：市面上翻译工具多为在线网页或独立客户端，本工具将三种输入方式（手动、截图、文件）与翻译引擎整合为桌面端一站式流程。特别是截图OCR翻译——用户看到任何屏幕上的外文，只需按一个热键框选即可获得翻译，无需切换到其他应用。

## 4. 系统架构设计

### 4.1 模块划分

```
ScreenCaptureOCR/
├── Form1.cs               # 主窗体：UI编排、翻译流程控制、文件解析
├── FormScreenshot.cs      # 截图遮罩窗体：全屏遮罩 + 鼠标拖拽选区
├── OcrService.cs          # 百度OCR API 客户端（token 管理 + 图片识别）
├── TranslateService.cs    # Google翻译 API 客户端
├── HistoryManager.cs      # 历史记录数据管理（增删、JSON序列化、缩略图存储）
├── HistoryItem.cs         # 历史记录数据模型
├── HistoryDetailForm.cs   # 历史详情弹窗（图片 + 原文 + 译文）
└── SettingsForm.cs        # 设置窗口（API Key / 热键 / 默认语言）
```

### 4.3 数据流说明

- **手动输入数据流**：`txtManualInput.Text` → `TranslateService.Translate()` → 更新 `rtbOcrResult`(原文) / `rtbTranslation`(译文) → `HistoryManager.Add()`
- **截图OCR数据流**：`CopyFromScreen` 捕获全屏 → `FormScreenshot` 选区交互 → `Bitmap.Clone` 裁剪 → `OcrService.RecognizeText()` 识别 → `TranslateService.Translate()` 翻译 → 更新文本框 + 保存历史
- **文件数据流**：`OpenFileDialog` 选取文件 → `ReadFileText()` 根据扩展名分发解析 → 文本填入 `txtManualInput` → 走手动输入流程
- **历史数据流**：每次翻译完成 → `HistoryManager.Add()` 构造 `HistoryItem` → 序列化为 `index.json` + 缩略图 `{id}.jpg`（仅截图来源）→ `RefreshHistoryList()` 刷新 ListView

## 5. 核心功能说明

### 5.1 三种输入方式

**手动输入翻译**：主界面提供文本框，用户直接键入或粘贴文字后点击"翻译"按钮。系统先判断是否有手动输入内容——若有则直接调用翻译引擎，跳过 OCR 环节，响应最快。

**截图OCR翻译**：支持 F1/F2/按钮三种方式触发。触发后主窗体最小化，调用 `Graphics.CopyFromScreen` 捕获全屏画面，弹出全屏遮罩窗体进行选区交互。通过 GDI+ 三层次绘制（底层截图 → 中层半透明遮罩 → 上层选区挖空）实现"聚光灯"视觉效果。选区坐标经 `DeviceDpi / 96f` 缩放因子变换后裁剪，适配高 DPI 屏幕。

**文件上传翻译**：点击"上传文本"按钮，通过 OpenFileDialog 选取文件，支持 .txt/.docx/.pdf 三种格式。.docx 通过解压 ZIP 提取 `word/document.xml` 中的 `w:t` 标签文本，.pdf 通过 PdfPig 逐页提取。解析结果自动填入手动输入文本框，复用已有翻译流程。

### 5.2 翻译引擎

通过 Google Translate 免费接口（`translate.googleapis.com`）实现翻译。`sl=auto` 参数自动检测源语言，支持中文/英语/日语/韩语/法语/德语/俄语 7种目标语言。选择免费接口降低了用户使用门槛——下载即用，无需注册申请任何 API Key。

### 5.3 OCR 识别（截图/图片输入的后端服务）

百度 OCR API 通用文字识别接口。通过 OAuth 2.0 客户端凭证模式获取 access_token 并缓存至过期前 5 分钟。图片自动 JPEG 压缩至最长边 ≤1024px，确保不超过 API 大小限制且传输效率最优。

### 5.4 历史管理

`HistoryManager` 类封装所有历史数据操作，从主窗体抽离实现数据与 UI 分离。每次翻译完成自动保存原文、译文、时间戳和来源类型（截图/手动/文件），截图来源同时保存 200×150 缩略图。保留最近 50 条记录，双击历史条目弹出详情窗口。

### 5.5 网络与异常处理

`OcrException` / `TranslateException` 携带 `IsNetworkError` 属性，区分网络故障与 API 业务错误。网络断开时弹窗提示"网络连接失败，请检查网络"，超时提示"请求超时，请检查网络"。API 业务错误（无效 Key 等）显示具体错误信息。

### 5.6 系统托盘与热键

`RegisterHotKey` Win32 API 注册全局热键，`WndProc` 拦截 `WM_HOTKEY` 消息触发截图。最小化时隐藏任务栏，显示 NotifyIcon 气泡提示。关闭时释放所有资源。

## 6. 应用环境

- **运行环境**：Windows 7+，.NET Framework 4.8
- **开发环境**：Visual Studio 2022，C# 7.3
- **外部依赖**：百度 OCR API（需免费注册获取 API Key），Google Translate API（免费接口，无需 Key）
- **限制条件**：OCR 和翻译依赖网络连接；百度 API 有 QPS 限制和 4MB 图片大小限制
